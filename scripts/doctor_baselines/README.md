# doctor_baselines — dispensor_doctor 실측 베이스라인

`dispensor_doctor.py`(구 senlyt_test_cli.py) 실측 원문 로그를 **물리 상태별 베이스라인**으로 보존한다.
나중에 같은 상태에서 다시 돌린 결과를 여기 파일과 비교하면(눈diff·타이밍 대조) **회귀·개선을 실측으로 판정**할 수 있다.

## 사용법 — 비교하는 법

1. 같은 물리 상태를 만들고 `dispensor_doctor.py ... run` 전체(8종)를 돌린다.
2. 결과를 아래 대응 베이스라인과 비교한다. 볼 축 3개:
   - **판정**(PASS/FAIL + errorCode) — 달라졌다면 동작 변경(의도된 것인지 확인)
   - **실패까지 시간** — 상태별 지문(아래 표). 자릿수가 바뀌면 실패 경로 자체가 바뀐 것
   - **전이 지연**(발행→queued→running) — 큐·SSE 축 건강도
3. 새 실측이 의미 있으면 같은 규칙(`YYYY-MM-DD-runN-{상태}.txt`)으로 파일을 추가해 시계열을 쌓는다.
